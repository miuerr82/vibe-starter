# flows.md

## 這份文件做什麼

- 用來定義此產品專案中的流程
- 流程是由已定義行為組成的有序步驟，用來描述事情如何一步一步發生
- 這份文件只負責排列行為順序，不重新定義角色或物件
- 流程中的每一步驟，都是將 `behaviors.md` 中已定義的行為串接起來，不在這裡重寫角色與物件參考

## 範例

### 任務建立流程

1. 執行建立任務
2. 執行指派任務
3. 執行通知負責人

### 申請審核流程

1. 執行送出申請單
2. 執行受理申請單
3. 執行審核申請單
4. 執行核准申請單
