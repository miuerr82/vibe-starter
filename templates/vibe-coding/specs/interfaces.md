# interfaces.md

## 這份文件做什麼

- 用來定義此產品專案對外的介面與互動面
- 這份文件可描述 API、事件、外部輸入、外部輸出，或其他資料交換接點
- 若後續出現系統之間怎麼互通、資料怎麼進出不清楚，應優先回到這份文件確認

## 不知道怎麼開始時

- 可以先從「有哪些外部系統、人或前端會和本系統互動」開始想
- 也可以先整理：
  - 誰呼叫本系統
  - 本系統會呼叫誰
  - 會交換哪些資料
- 若還是不確定，可先回頭看 `boundaries.md`、`flows.md`、`objects.md`，把有跨系統或跨邊界的接點整理成介面

### 可直接請 AI 協助的 Prompt

> 請根據我的 `boundaries.md`、`flows.md`、`objects.md`，幫我整理出此專案應該寫進 `interfaces.md` 的介面。請區分 API、事件、外部輸入、外部輸出，並盡量用高層方式描述，不要直接變成實作文件。

## 範例

### API 介面

- 任務建立 API

### 事件介面

- 使用者上傳檔案

### 外部輸出

- 匯出報表資料
