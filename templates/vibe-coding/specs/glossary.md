# glossary.md

## 這份文件做什麼

- 用來定義此產品專案中的關鍵術語，避免不同文件出現同名不同義或同義不同名
- 這份文件只處理詞彙與定義，不描述流程、狀態或規則
- 若其他 spec 文件出現容易混淆的詞，應優先回到這份文件統一說法

## 不知道怎麼開始時

- 可以先從 `0_project.md`、`objects.md`、`behaviors.md` 裡反覆出現的詞開始整理
- 若同一個意思出現多種叫法，也應收進來統一
- 若不確定某個詞算不算術語，可以先看它是否會在多份文件中反覆出現並影響理解

### 可直接請 AI 協助的 Prompt

> 請根據我的 `0_project.md`、`objects.md`、`behaviors.md`、`flows.md`，幫我整理出此專案應該寫進 `glossary.md` 的術語表。請統一重複或相近的詞，輸出為「術語 / 定義 / 備註」表格，並避免把流程步驟直接寫成術語。
